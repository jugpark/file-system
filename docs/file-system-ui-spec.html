01

개요

목적. 소규모 팀이 로컬 서버에 쌓아 둔 파일을, 웹 브라우저에서 탐색·업로드·공유·권한 관리까지 처리할 수 있게 하는 것이 이 인터페이스의 역할입니다. 별도 클라이언트 설치 없이 링크 하나로 접근하고, 누가 무엇을 언제 바꿨는지 화면에서 바로 확인합니다.

인증 · 권한

Discord OAuth2로 로그인하고, 서버의 Role을 그대로 접근 권한으로 사용합니다.

Discord OAuth2 · Server Roles

파일 관리

Node.js의 fs 모듈로 로컬 스토리지의 실제 파일을 읽고 씁니다.

Node.js · local storage

메타데이터 · 로그

작성자, 수정 시각, 권한 등 파일에 얽힌 기록을 내부 DB에 남깁니다.

SQLite / PostgreSQL
02

전체 화면 레이아웃

Google Drive·Nextcloud와 닮은 3단 레이아웃을 채택합니다. 좌측 사이드바, 중앙 메인 뷰, 우측 정보 패널이라는 익숙한 구조를 따라 학습 비용을 최소화합니다. (위 미리보기가 완성 형태의 축약입니다.)

A좌측 사이드바 — LNB & Tree

탐색의 출발점. 권한에 따라 보이는 공간이 달라집니다.

  • 상단 메인 액션 — 회사 로고와 함께 ‘새로 만들기(업로드 · 폴더 생성)’ 버튼을 가장 눈에 띄게 배치합니다.
  • 권한 기반 네비게이션 — 내 작업 공간(개인 전용), 부서별 공유 폴더(디자인팀·개발팀 등 Role로 접근 가능한 폴더만 노출), 최근 파일(DB 로그 기준 최근 수정 항목).
  • 트리 뷰 — 아코디언으로 폴더를 접고 펼치며, 하위 폴더로 빠르게 이동합니다.

B상단 네비게이션 — GNB

현재 위치를 알려주고, 원하는 파일을 곧장 찾게 합니다.

  • 경로 탐색(Breadcrumb) — 현재 위치를 보여 주고, 상위 폴더로 한 번에 이동합니다.
  • 검색바 — 파일명·확장자를 DB 인덱스로 검색합니다.
  • 유저 프로필 — Discord 아바타·닉네임과 로그아웃 버튼.

C중앙 메인 뷰 — Explorer

파일을 실제로 다루는 작업 공간.

  • 뷰 전환 토글 — 리스트(테이블) ↔ 그리드(썸네일·아이콘)를 오갑니다.
  • 드래그 앤 드롭 존 — 바탕화면에서 파일을 끌어다 놓으면 즉시 업로드하며, 놓을 수 있는 영역을 시각적으로 피드백합니다.
  • 리스트 뷰 컬럼 — 파일/폴더명(아이콘 포함), 마지막 수정자, 수정 일시, 파일 크기.

D우측 정보 · 로그 패널

파일이나 폴더를 한 번 클릭하면 열리는, DB와 맞물린 핵심 패널.

  • 기본 정보 — 파일명, 포맷, 크기, 저장 경로.
  • 권한 상태 — 지금 로그인한 유저의 Role 기준 권한을 배지로 표시(읽기 전용 · 수정 가능).
  • 활동 로그 — 파일의 생애주기를 타임라인으로 보여 줍니다.

핵심 상호작용 · UX 흐름

3.1  로그인

복잡한 폼 없이, 웰컴 메시지와 [Discord로 로그인] 버튼 하나로 끝냅니다. 인가되지 않은 유저가 접속하면 사내 디스코드 서버 멤버만 접근할 수 있습니다 에러 모달을 띄웁니다.

3.2  업로드 · 다운로드

업로드. 파일을 드래그 앤 드롭하면 우측 하단에 진행률 토스트가 뜨고, 완료 시 업로더의 Discord ID와 시각을 DB에 기록합니다. 다운로드. 더블 클릭하거나 우클릭 메뉴에서 ‘다운로드’를 선택합니다.

3.3  우클릭 메뉴

파일·폴더에 우클릭하면 OS와 같은 컨텍스트 메뉴가 열립니다. ‘이름 바꾸기’와 ‘삭제’ 같은 동작은 수정 권한을 확인한 뒤 실행하고, 삭제는 곧바로 지우지 않고 휴지통으로 옮기며 삭제자를 DB에 남깁니다.
